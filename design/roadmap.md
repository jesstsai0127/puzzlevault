# 設計計劃書 v1 —— 找出「好玩」是什麼

> 這份文件的目的：在畫任何一張圖之前，先用最便宜的方式（純文字符號）確定這個遊戲的核心玩法好不好玩。
> 開發紀律：**一次只驗證一個元素**，做出可玩的 MVP → 實際玩 → 決定「留或砍」→ 才做下一個。
> 視覺完全不重要，前期一律用文字/符號呈現，畫面旁邊直接用中文寫規則，讓任何測試者 30 秒內看懂怎麼玩。

---

## 第一章 · 怎麼玩（最重要的一章）

> 這一章決定玩家願不願意玩。如果一個新玩家看完這章、看一眼畫面還是不懂要幹嘛，那就是我們失敗了——舊版就是敗在這裡（測試者一直說「無法理解怎麼玩」）。所以這一章要做到：講給完全沒玩過的人聽，他也能立刻懂。

### 一句話說明
**你指揮兩名俠客，守住中央的「陣」。敵人一波波衝過來想打爆陣，但它們下一步要做什麼會「事先全部公開」，你要在出手前想清楚：怎麼走位、推擋、出招，讓這一波的攻擊全部落空。撐過所有波次，陣還活著，你就贏了。**

不需要殺光所有怪——只要它們打不到陣，就算它們活著也沒關係。這是重點：**這是一個「防守解謎」遊戲，不是「清怪」遊戲。**

### 畫面上你會看到什麼（全部用文字符號）
```
牆牆牆牆牆牆牆牆
牆 焰 ·  ·  ·  鬼 牆
牆 陣 ·  淵 ·  鬼 牆
牆 晴 ·  ·  ·  ·  牆
牆牆牆牆牆牆牆牆
```
- `焰`＝你的近戰俠客（李焰）、`晴`＝你的遠程俠客（蘇晴）
- `陣`＝你要守的基地，有血量，被打到就掉血，歸零就輸
- `鬼`＝來犯的妖物，頭上會有箭頭預告它下一回合的動作
- `淵`＝深淵地格，被推進去的怪直接消失
- `牆`＝擋路的牆
- 畫面右側固定有一塊**中文規則面板**，隨時寫著「現在該做什麼」

### 一個回合實際怎麼進行（走一遍給你看）
1. **看預告**：回合一開始，每隻 `鬼` 頭上就亮出箭頭——比如兩隻鬼都顯示「↓ 往陣的方向移動一步」。你現在就知道它們待會要幹嘛，沒有任何隱藏、沒有骰運氣。
2. **想解法**：你有一份**兩人共用的移動點**，加上每個俠客**自己的真氣（MP）**可以出招。你盤算：
   - 用 `晴` 的飛劍術射掉靠最近那隻鬼？
   - 用 `焰` 走到陣前面那格，用身體把通道堵死？
   - 還是用排雲掌一掌把逼近的鬼**推進 `淵`**、一次解決？
3. **試與反悔**：你可以隨便試，送出回合前都能「復原」重來，像在解一道題。
4. **結束回合**：你按下結束，敵人**就照剛才預告的動作執行**（不會突然改變）。你立刻看到結果——這一波有沒有讓陣掉血。
5. **下一波**：撐過這波，下一波來。守到最後一波、陣還活著＝**贏**。

### 為什麼會好玩（我們要驗證的假設）
- **每一回合是一道有解的謎題**：資訊全公開，一定有一組正確走法能讓陣不掉血，考的是你算不算得出來——這是「解謎」的樂趣。
- **想清楚就贏、想錯就付代價**：沒有運氣成分，輸了是你算錯，不是被骰子害的——這種「我剛剛那步好聰明」的頓悟感是核心爽點。
- **推擋比殺戮更聰明**：把怪推進淵、用身體卡位、改變它們的路徑，往往比硬殺更省資源——這讓走位跟推擊變成主角。

### 為什麼會一直想玩下去（後期才加，但先講清楚方向）
- **同一關有不同結局**：速戰速決 / 零失血完守 / 慘勝，會給不同獎勵，讓你想重打拿更好的成績。
- **帶著養成往下打**：這一關拿到的獎勵能強化下一關的俠客，一整場（run）像在滾雪球。
- **每次都不一樣**：同一關每次進去，敵人的組成/位置會有隨機變化，不會背版。

## 第二章 · 設計支柱（所有決策都不能違背這幾條）

1. **回合是謎題，一整場是養成**：單一回合＝資訊全公開的確定性謎題（解謎樂趣）；一整場遊戲＝隨機生成、帶著獎勵往下打的 roguelite（重玩樂趣）。這兩層分開，才能同時要到「動腦」跟「爽度」。
2. **守陣，不是清怪**：勝利＝保護基地，不是消滅敵人。這是 Into the Breach 的真正核心（它守的是電網/建築，不是殺光蟲族）。
3. **資訊全公開，零運氣**：敵人下一步永遠事先預告，沒有命中率、沒有暗骰。輸了永遠是玩家自己算錯。
4. **攻守天生兩面**：守基地這個框架本來就有「攻擊方」跟「防守方」兩個座位。現在攻擊方由 AI 坐，將來可以換真人坐（見第六章），但**現在就要為兩面設計**。
5. **文字優先、玩法優先**：前期一律文字符號＋畫面旁中文規則，先確定好玩，美術最後才碰。

## 第三章 · 開發策略：一次一個元素

每個階段只加**一個**新元素，做成能玩的 MVP，實際部署到 GitHub Pages 讓使用者玩，然後回答那個階段的**驗證問題**，決定**留或砍**，才做下一階段。這樣任何時候「變好玩」或「變難玩」都能歸因到單一改動。

每個階段共通做法（講一次，之後不重複）：
- 全程文字符號，不做美術
- 畫面旁固定一塊中文規則面板，讓測試者秒懂
- 做完就 build 部署到 https://jesstsai0127.github.io/puzzlevault/ 讓使用者實玩
- 使用者給「留或砍」，我才動下一階段

## 第四章 · 主線（骨幹四階段）

### Phase 1 — 守陣（勝利條件翻轉）
- **唯一新元素**：勝利＝守住基地，不是殺光怪。怪物衝向 `陣`，碰到就扣陣的血；陣血歸零＝輸；撐過 N 波＝贏。
- **MVP**：文字符號盤面 + 中文規則面板。怪直線衝陣，玩家用移動卡位＋排雲掌把怪推離/推進淵。
- **驗證問題**：「守住陣線、把怪推開」這個核心迴圈本身好不好玩？
- **留或砍**：如果連推怪卡位都無聊 → 整個方向要重想；如果會想「再守一波」→ 往下走。
- **Phase 1 攻擊方目標（採用建議預設，開工前可再確認）**：怪物**只衝陣、無視英雄**——讓謎題最乾淨、最好驗證。之後要加「也會攻擊英雄」很容易。

### Phase 2 — 解謎化（每一波是一道有解的題）
- **唯一新元素**：手工設計的謎題波次，保證「這一波一定有一組操作能把傷害完全擋掉」，資源故意剛好夠。
- **MVP**：3 道手工關卡。
- **驗證問題**：「每波是一道設計過的謎題」比「隨機衝過來」好玩嗎？
- **留或砍**：如果亂玩也能過 → 謎題張力不夠，收緊資源或加威脅。

### Phase 3 — 多結局評級（快慢回合不同結局）
- **唯一新元素**：同一關依表現給不同結局。例：`≤3 回合速勝` / `零失血完守` / `慘勝`，各給不同的具名獎勵（此階段先只顯示，不影響後續）。
- **MVP**：一關結束跳出評級與對應獎勵。
- **驗證問題**：「同一關能追求不同結局」有沒有讓人想重打拿更好的？
- **留或砍**：沒人想重打 → 評級無用，砍掉。

### Phase 4 — 累加養成（把獎勵串成一場 run）
- **唯一新元素**：Phase 3 的獎勵帶進下一關（強化技能 / 永久 buff / 多一個俠客）。
- **MVP**：3 關迷你戰役，上一關評級讓你三選一個 buff 用在下一關。
- **驗證問題**：「帶著累積效果往下打」有沒有讓整體更黏？
- **留或砍**：帶不帶都沒差 → buff 不夠有感，放大效果。

## 第五章 · 深度待辦池（骨幹證明好玩後，才一次拉一個進來）

順序可調，每一個仍遵守「一次只加一個、加完就問還好不好玩」：
- **地形效果**：全體中毒、地格陷阱、火/毒持續傷害
- **玩家佈置陷阱**：Orcs Must Die 那套，一個全新的「動詞」
- **怪物屬性/元素相剋**，且**開戰前預告屬性**
- **更多俠客與技能、平衡調校**

## 第六章 · 連線對戰（設計於前，實作於後）

**方向**：不對稱攻防——一方守陣、一方派怪進攻。守基地這個框架天生兩面，回合制又讓連線變成最便宜的那種（輪流傳操作，不需要即時 netcode）。

**但實作放到最後**，因為它會：(1) 把單機翻成連線、要開始養後端；(2) 打破現在零維運的免費靜態部署；(3) 在核心迴圈還沒證明好玩前就做，正好踩進「還沒確認方向就燒額度」的坑。

**現在就做的（幾乎零成本的保險）**：把「攻擊方」設計成一個可替換的**角色（role）**——就是一組決策：派哪隻怪、哪個生成點、哪個時機。單機時 AI 坐這個座位（＝本來就要做的波次/AI 系統），將來真人透過網路坐同一個座位，引擎幾乎不用動。**先用 AI 攻擊方驗證「防守好不好玩」**——對 AI 不好玩，netcode 也救不了。

**真要早點嚐 PvP**：最便宜的是「同機輪流」或「分享房間碼的非即時對戰」，不開即時 server。

**最貴、放最後**：依遊玩經驗配對（skill-based matchmaking）——它會拖進帳號系統、使用者資料庫、對戰記錄儲存，跟「本機優先、隱私優先」原則最衝突。隨機配對比它便宜得多。

## 第七章 · 架構原則（保留 vs 打破）

**保留**（已驗證、是上面所有玩法的好地基）：格子系統、回合狀態機、確定性意圖預告、效果動詞（damage/push/shield/heal）、資料驅動 JSON、既有測試。

**打破**（這三個「舊假設」綁死了新方向）：
- 勝利＝殺光 → 改成勝利＝守住基地
- 固定兩個英雄 → 改成可組隊、可養成
- 現有波次格式 → 改成「攻擊方 role」產生的攻勢（AI 或真人都能填）

**新增原則**：
- **攻守分離**：攻擊方是可替換的 role，不寫死成 AI。
- **種子生成**：同一關每次用不同 seed 產生變化（強化 roguelite 重玩性 + 將來 PvP 變化性）。
